# 校线质量异常分析大屏图表群设计方案

### 核心分析板块

#### 第一板块：全局概览与趋势（看大盘）



**业务目的：** 快速了解当前质量管控的整体盘面和问题发生的节奏。



- 1. [核心指标卡] 质量大盘数据卡（KPI Cards）： 顶部展示核心数字，如总异常数、未闭环数、平均处理耗时(小时)、内部/外部责任占比。

  

  

- 2. [时序图] 质量异常每日提报趋势（折线图）： X轴为时间（按天/周），Y轴为异常数量。用于揭示质量问题是否存在集中爆发的周期或时间节点（如某批次物料集中上线时）。

  

  

- 3. [状态监控] 异常单据处理状态分布（漏斗图 / 环形图）： 展示“提报 -> 诊断中 -> 整改中 -> 验收合格”的转化留存，直观暴露当前积压在哪个环节。

  

  

#### 第二板块：痛点对象定位（找重灾区）



**业务目的：** 回答“问题主要发生在什么车、什么部位”的问题。



- 4. [项目级] 各项目异常贡献度（饼图）： 展示不同项目（如 DUATBEL、SHL11D73XF）的故障占比，决定资源倾斜方向。

  

  

- 5. [列车级] 故障频发列车 Top 15（水平条形图）： X轴为故障数，Y轴为具体列车号/节车号，精准定位出哪几列车是“病号车”。

  

  

- 6. [构型级] 异常多发产品构型（矩形树图）： 面积大小代表数量，直观展示终端、线号、标签、电缆等各类构型的故障密集度。

  

  

- 7. [交叉分析] 核心项目 vs 构型热力图（热力图）： X轴为构型，Y轴为项目，颜色深浅代表故障数，直接暴露出特定项目在特定构型上的交叉痛点。

  

  

#### 第三板块：缺陷特征剖析（查病理）



**业务目的：** 回答“具体出了什么错”的问题。



- 8. [现象级] 现象分类占比（南丁格尔玫瑰图）： 展示诸如“点位不符图”、“图文歪斜/模糊”、“文本错”等现象的分布情况。

  

  

- 9. [模式级] 核心失效模式排行（帕累托图）： 找出20%导致了80%故障的核心失效模式，进行二八定律分析。

  

  

- 10. [因果链] 失效现象与原因的演变路径（桑基图）： 展示从“现象分类”流向“失效模式”再到“失效原因”的数据流，看穿各种表象背后的核心根因。

  

  

#### 第四板块：责任归属与追溯（定责问责）



**业务目的：** 清晰界定责任归属，指导绩效考核与供应商管理。



- 11. [内外对比] 责任分类分布（极坐标图 / 饼图）： 宏观对比“制造”、“供应商”、“设计”等责任维度的比例。

  

  

- 12. [黑榜通报] 责任单位/班组缺陷红黑榜（柱状图）： 曝光制造班组、机车事业部管线班以及各供应商的异常数量排名。

  

  

- 13. [责任交叉] 责任单位 vs 失效原因（堆叠柱状图）： X轴为责任单位，内部堆叠不同失效原因，对比辨析不同单位的核心错误类型。

  

  

#### 第五板块：流程效率与时效评估（抓瓶颈）



**业务目的：** 评估团队响应速度和流程阻塞点。



- 14. [整体时效] 异常处理耗时分布（直方图）： 展示耗时的区间分布（如0-24h、24-72h、>100h的长尾异常占比）。

  

  

- 15. [班组对标] 各班组平均处理时效对比（气泡图 / 散点图）： X轴为处理工单量，Y轴为平均耗时，气泡代表不同班组，快速识别低效班组。

  

  

#### 第六板块：整改方案与经验挖掘（防重现）



**业务目的：** 通过文本挖掘发现更深层的问题。



- 16. [诊断归类] 初步诊断结果聚类（词云 / 树图）： 揭示系统填报的初步诊断集中在哪些关键词上。

  

  

- 17. [文本挖掘] 现象详情与整改方案高频词汇（双面对比词云）： 提取员工填写的自由文本关键词，对比“现象”与“整改”高频词，评估整改措施是否单一。

  

  

#### 第七板块：人员质量红黑榜（抓落实与培训）



**业务目的：** 回答“谁总在犯错及短板所在”，为人员绩效考核和精准培训提供数据支撑。



- 18. [人员问责] 核心责任人缺陷分布排行榜（堆叠柱状图 / 下钻树图）： Y轴按总数倒序排列责任人姓名，柱子内部使用失效模式分层堆叠；支持交互悬浮查看个人专属的“失效模式成分表”。

  

  

- 19. [班组病理图] 生产工位/班组核心缺陷分布（雷达图 / 矩阵气泡图）： 维度为所属班组，指标为各类失效模式；通过图形凸出的尖角识别该班组特有的痛点。

### 🚀 新增扩展业务维度

#### 扩展维度一：空间拓扑与物理位置追踪（找寻空间环境死角）



**业务思考：** 质量问题是否和线缆所在的物理环境有关（如走线空间紧凑、距离过长）。



- 20. [列车空间热力图] 各节车号/物理位置的缺陷密度分布： 提取校线节车号和产品位置，使用列车结构模拟图直接标出故障率最高的车厢区域。

  

  

- 21. [走向连线图] 线路起始点与终止点的错误网络（和弦图）： 分析起始与终止位置的关联，识别跨柜线缆的图纸设计盲区。

  

  

#### 扩展维度二：异常类别溯源（区分“操作背锅”与“设计缺陷”）



**业务思考：** 用数据把“一线制造问题”和“上游设计问题”剥离开。



- 22. [本源漏斗图] 异常类别构成占比（饼图 / 环形图）： 统计异常类别；若“设计问题”占比过高，则需启动技术图纸专项复核。

  

  

- 23. [工序脆弱点] 不同工序/工步的缺陷爆发率（横向漏斗图）： 结合工序字段，定位在布线、穿管、压接或端子嵌入等具体哪个步骤出错最多。

  

  

#### 扩展维度三：检验/诊断效能与负载（评估“质检防线”是否可靠）



**业务思考：** 评估质检员或工艺员的工作负载，查找单据积压的节点。



- 24. [人员负载图] 诊断人/审核人的业务压力与时效对比（气泡图 / 散点图）： 找出名下挂着大量异常单据且处理时间极长的“瓶颈节点人物”。

  

  

- 25. [火眼金睛榜] 提报人发现的隐患排行（柱状图）： 提取提报人姓名，表彰发现问题最多、最准的质检员与测试员。

  

  

#### 扩展维度四：时间节律与疲劳曲线（洞察“疲劳作业”规律）



**业务思考：** 观察质量问题是否在特定时间段（如快下班时或夜班期间）爆发。



- 26. [时间节律图] 异常发生时间段热力图（日历热力图 / 24小时时钟图）： 提取时间戳并精确到小时进行聚合，识别因人员疲劳导致的高发期，以便增加巡检频次。

  

  

#### 扩展维度五：整改代价与图文闭环率（评估整改质量）



**业务思考：** 评估处理问题的真实难度与合规严谨度（是否有图有真相）。



- 27. [证据留存率] 带有“返工图片/故障图片”的单据比例（仪表盘）： 评估质量闭环的严谨度，排查“走过场”的合规性风险。

  

  

- 28. [整改动作解构] 返工依据分类分析（词云 / 树图）： 分析整改描述和依据，比对走正规“设计变更（ECN）”处理与车间私下“灵活处理”的比例。
